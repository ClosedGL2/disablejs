Ways JavaScript is Overused

Requiring JavaScript to view the page at all

If you view the HTML source of Quora's home page, you will see that it starts out with a bunch of scripts, a "Please enable JavaScript" message, a hidden error message, and abolutely nothing else.

Many webpages may have no consideration for clients with JS disabled, most commonly with loading screens that will never go away without JavaScript.

It's best to serve all the different parts of the page at once instead of making the client piece together the page, creating unnecessary overhead for the client.

Using JavaScript to change CSS for animations

Using JavaScript for animations is extremely resource intensive. The browser has to execute the script changing the CSS, and completely recalculate so many elements every frame.

It's better to use CSS animations.

The worst case of this I've seen is when the whole page fades in by changing the opacity of everything. When JavaScript is disabled, everything is there but I can't see anything!

Smooth scrolling or overriding scrolling in any way

Leave my scrolling settings alone ffs!

<a href="#" onclick='window.location="page.html"'>

When hovering the mouse over a link, browsers will normally show where the link goes in the status bar or the bottom left corner.

Using JavaScript instead of <a href="page.html"> makes it so visitors can't see where links go, and makes it impossible to use without JavaScript.

Endless scrolling

Automatically loading more content when scrolling to the bottom of the page can make it impossible to reach links at the bottom of a page, eats up the client's resources, and wastes the visitor's time constantly looking at new content.

Data collection and analytics scripts

Not only is this a massive invasion of privacy, but it eats up bandwidth and creates more overhead for the client.

Megabytes worth of JavaScript

Running lots and lots of JavaScript at once can really slow down a webpage, especially when using browsers like Pale Moon and SeaMonkey that run JavaScript slower than Chrome and Firefox.

I wouldn't blame such browsers for running JavaScript slowly. A normal browser shouldn't have to account for lots of bloat.
